# Runbook: Deploy Wazuh SIEM

Stand up the all-in-one Wazuh manager + indexer + dashboard VM on the
SecretCon Proxmox host. Target IP `192.168.61.10`, VMID `110`.

## Prerequisites

- SSH access to the Proxmox host as `root@192.168.60.1`.
- WireGuard tunnel up; routes for `192.168.60.0/24` and
  `192.168.61.0/24` present on the workstation.
- An Ubuntu 22.04 cloud-image template at VMID `9000`. If missing, run
  `build-wazuh-template.sh` first.
- A clone of this repo on the workstation. The deploy and verify
  scripts SSH into the host themselves.

## Steps

### 1. Build the base template (one-shot, first run only)

```
ssh root@192.168.60.1 'bash -s' < scripts/proxmox/build-wazuh-template.sh
```

This creates a Proxmox template at VMID `9000` from the Ubuntu jammy
cloud-image. Skip if already present (`qm list | grep 9000`).

### 2. Deploy the SIEM VM

```
bash scripts/proxmox/deploy-wazuh-siem.sh
```

What this does:

- Destroys any existing VMID `110` after confirming with the operator.
- Clones template `9000` into VMID `110`, name `wazuh-siem`.
- Attaches two NICs: `net0` on `vmbr0` (DHCP) for egress, `net1` on
  `vmbr1` static `192.168.61.10/24` for agent traffic.
- Seeds cloud-init with the NoCloud payload at
  `provisioning/cloud-init/wazuh/user-data`.
- Boots, waits for SSH, then runs
  `provisioning/bash/bootstrap-wazuh-ubuntu.sh` which installs Wazuh
  via the upstream `wazuh-install.sh -a` script and seeds the `ews`
  agent group plus Suricata rules `86600`-`86604`.

Expected runtime: about 20 to 30 minutes if the template exists, plus
template build time on the first run.

### 3. Verify

```
bash scripts/proxmox/verify-wazuh-siem.sh
```

Checks:

- Manager port `1514/tcp` (agent) open and accepting.
- EVE listener `1514/tcp` (JSON) open.
- Dashboard `443/tcp` open.
- Agent group `ews` present.
- Custom rules `86600` through `86604` in
  `/var/ossec/etc/rules/local_rules.xml`.

A synthetic EVE alert sent to TCP/1514 should appear in the dashboard
as rule `86601`. The verify script can be extended to test this end
to end.

### 4. Access the dashboard

If the workstation cannot route directly to `192.168.61.0/24`, tunnel:

```
ssh -N -L 8443:192.168.61.10:443 root@192.168.60.1
```

Open `https://localhost:8443`. Accept the self-signed certificate.

The dashboard admin password is generated by `wazuh-install.sh` and
printed at the end of the bootstrap. Capture it on first deploy. It is
not stored in this repo.

## Rollback

```
ssh root@192.168.60.1 'qm stop 110 && qm destroy 110 --purge'
```

The template at VMID `9000` is untouched and can be redeployed at any
time.

## Common failures

- "Cloud-init never ran": the seed ISO did not attach. Run
  `qm config 110 | grep ide2` on the host. If empty, re-run the
  deploy script with `--force`.
- "Bootstrap stalled": check `/var/log/cloud-init-output.log` inside
  the VM via SSH. The Wazuh installer is large and slow on first run,
  but it should not stall for more than 10 minutes silently.
- "Agent group `ews` missing in dashboard": the bootstrap ordering
  failed. Re-run `/usr/local/sbin/seed-wazuh-extras.sh` inside the VM
  to recreate the group and rules.
